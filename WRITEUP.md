## What it is

A web version of **Jeopardy!** reimagined as a *daily, self-writing* game. Each board
has six categories of five clues. Most are drawn from a real 538,000-clue Jeopardy!
archive; one — **TODAY'S HEADLINES** — is composed on the spot by an LLM from the
day's actual news. You pick a tile, type a free-text response, and an AI host rules
on it and reacts in character. There's a Daily Double with wagering, score tracking,
and a shareable end-of-game score.

**Audience:** trivia fans and casual players who want a quick, fresh daily game — the
"same board for everyone today, come back tomorrow" loop that made Wordle sticky.

## How AI is used (meaningfully, in two places)

1. **AI as referee.** Real Jeopardy's hard problem is judging free-text: is *"FDR"*,
   *"Roosevelt"*, or *"what is Franklin Delano Roosevelt"* correct? String matching
   can't. A Claude **Haiku** call grades each answer leniently — accepting misspellings,
   missing articles, and unambiguous partials, ignoring "question phrasing" — and
   returns a verdict, a one-line reason, and an in-character host quip. This *is* the
   gameplay; it cannot exist without an LLM.
2. **AI as writer.** The live category is generated by Claude **Sonnet** from real
   headlines, producing five fair, difficulty-ascending clues with answers. This is
   what makes the game current and genuinely different every day.

A no-key fallback (local fuzzy matching, archive-only board) keeps the app playable
and demoable without credentials.

## How data is used (at scale)

The [538k-clue dataset](https://github.com/jwolle1/jeopardy_clue_dataset) (1984–2025)
is streamed through `scripts/build-clues.mjs`, which **reconstructs intact categories**
by grouping clues by air-date + round + category, keeps only those with a clean
five-rung value ladder, strips media-dependent ("seen here") clues and markup,
dedupes by name, drops dry pure-year categories, and **deterministically samples 600
topically diverse categories** (~440 KB) committed to the repo. So the data isn't a
flat trivia bank — it's used to deliver *authentic, structurally-correct Jeopardy
boards*, normalized to standard $200–$1000 values regardless of era.

Live data comes from two free, no-auth sources — the Hacker News (Algolia) front page
and the Wikipedia featured feed ("in the news" + most-read) — merged into the prompt
that writes the headlines category.

## Dynamic behavior

- **Daily freshness:** the headlines category is regenerated each UTC day from live
  news (cached per day so a day's board is stable and cheap).
- **Per-player adaptation:** the client tracks lifetime accuracy in `localStorage`;
  the board API takes that skill level and **shifts the standard/hard category mix**,
  so stronger players get harder boards over time.
- **Replayability:** boards are seeded by `(date, skill, seed)`, so they're reproducible
  yet a "New board" gives a fresh, deterministic layout.

## Key design & architectural decisions

- **Next.js 16 / React 19 on Vercel** — one repo for UI + serverless API, instant
  public URL, zero infra. Tailwind v4 for a fast, authentic board UI.
- **Two-model split** — cheap/fast Haiku for the per-answer hot path (with prompt
  caching on the system prompt), higher-quality Sonnet for the once-a-day writing
  task. Keeps latency and cost low.
- **Precompute the corpus at build time, not request time** — the 77 MB source is
  distilled offline into a small committed JSON, so gameplay needs no DB and the
  board endpoint is instant.
- **Graceful degradation** — every AI path has a deterministic fallback, so a missing
  key or a flaky news API never breaks the game.
- **Scope tradeoffs (≈3 hrs):** shipped the core loop, real corpus, LLM judge, live
  category, Daily Double, adaptivity, and share. Deliberately deferred: a shared-KV
  daily board + global leaderboard, host TTS voice, and multiplayer — all noted as
  next steps rather than half-built.

## Tools used

Next.js 16, React 19, TypeScript, Tailwind CSS v4, the Anthropic SDK (Claude
